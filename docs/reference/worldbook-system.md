# SillyTavern 世界书系统深度解析 (World Info / Lorebook)

> 本文档解析世界书条目的激活逻辑、权重计算以及高级定时效果（Timed Effects）。

## 1. 激活逻辑与扫描流程

世界书通过扫描聊天上下文（Haystack）来查找关键字（Needle）。

### 1.1 扫描范围 (Scanning Buffer)
扫描缓冲区不仅包含聊天记录，还包括：
- `personaDescription`: 用户人设。
- `characterDescription`: 角色描述。
- `characterPersonality`: 角色性格。
- `characterDepthPrompt`: 深度提示词。
- `scenario`: 场景设定。
- `recurseBuffer`: 递归触发的内容（如果开启了递归扫描）。

---

## 2. 定时效果 (Timed Effects)

这是世界书中最高级的逻辑部分，用于模拟状态的变化（如中毒、冷静期）。

### 2.1 Sticky (粘性激活)
- **字段**: `sticky` (数字，代表消息数)。
- **效果**: 一旦条目被激活，它会在接下来的 N 条消息中保持激活状态，即使扫描缓冲区中不再包含关键字。

### 2.2 Cooldown (冷却时间)
- **字段**: `cooldown` (数字)。
- **效果**: 在 Sticky 状态结束后，条目会进入 N 条消息的冷却期。在此期间，即使关键字出现，条目也不会再次激活。

### 2.3 Delay (延迟激活)
- **字段**: `delay` (数字)。
- **效果**: 聊天记录必须达到至少 N 条后，该条目才开始参与扫描。常用于剧情的阶段性解锁。

---

## 3. 权重与逻辑控制 (`selectiveLogic`)

条目可以通过“次要关键字”实现复杂的逻辑判断：
- **AND_ANY (0)**: 只要有主要关键字就激活，次要关键字存在会增加分数。
- **AND_ALL (3)**: 必须主要关键字存在，**且所有**次要关键字都存在才激活。
- **NOT_ANY (2)**: 主要关键字存在，但**任何**次要关键字存在则不激活。

---

## 4. 深度与组装参数

激活的条目会根据其配置注入 Prompt：
- `depth`: 注入的深度（决定在上下文中的顺序）。
- `probability`: 激活概率（0-100）。
- `order`: 组装顺序（当多个条目在同一深度时，Order 越大越靠前）。

---

## 5. 开发者建议

1. **关键词优化**: 优先使用正则表达式以减少误触。
2. **递归调用**:慎用递归扫描，它会显著增加 Token 消耗。
3. **元数据利用**: 利用 `extra_data` 存储条目的额外说明或创作者备忘录。
